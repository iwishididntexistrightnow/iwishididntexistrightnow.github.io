Project Genesis: Subversion Catalyst - Learning Trajectory

User Profile Summary: No formal programming experience, strong pre-calc math (needs factoring practice), basic Gentoo install comfort (no scripting/maintenance), conceptual ML understanding, high-intensity/rigor/density learning style (action/project-based, associative), expects struggle but needs low friction, cost no object, direct instruction preferred, prioritize low-latency tech choices.

Target Roles (Initial): System Administrator / Programming Associate @ NC Target Companies.

Primary Languages (Initial Focus): C (Mastery), Rust (Foundational), Bash (Scripting).

Module 0: Laying the Cornerstone - Computation, Tools, and C Fundamentals

Rationale: This module establishes the absolute bedrock: core mathematical tools, the Gentoo Linux environment, essential developer tooling (Vim, Git, Bash), fundamental computational concepts, and the C programming language. Rigorous C mastery is prioritized early as it underpins nearly all low-level systems work. Tooling is introduced immediately for practical application.

Glossary

ABI (Application Binary Interface)
The low-level interface between an application program and the operating system or another application.
Arena Allocator
A memory allocation strategy where a large block of memory (the arena) is allocated upfront, and subsequent smaller allocations are carved out from this block, often with faster allocation/deallocation within the arena's lifetime.
BSP (Board Support Package)
Software containing hardware-specific drivers and other routines that allow a particular operating system to function in a particular hardware environment (common in embedded systems).
Cache Coherence
Ensuring that multiple caches in a multi-processor system maintain a consistent view of memory.
Context Switch
The process of storing the state of a process or thread so that it can be restored and resume execution later, allowing another process/thread to run.
DPDK (Data Plane Development Kit)
Libraries and drivers for fast packet processing, often bypassing the kernel's network stack.
DWARF
A standardized debugging data format used by many compilers and debuggers to support source-level debugging.
ELF (Executable and Linkable Format)
A common standard file format for executables, object code, shared libraries, and core dumps.
FPGA (Field-Programmable Gate Array)
An integrated circuit designed to be configured by a customer or a designer after manufacturing – hence the term "field-programmable".
GDB (GNU Debugger)
A portable debugger that runs on many Unix-like systems and works for many programming languages, including C, C++, and Rust.
Gentoo
A source-based Linux distribution known for its flexibility, performance potential, and steep learning curve, forcing users to understand system configuration deeply.
IPC (Inter-Process Communication)
Mechanisms an operating system provides to allow processes to manage shared data and communicate with each other.
ISA (Instruction Set Architecture)
The part of the computer architecture related to programming, including the native data types, instructions, registers, addressing modes, memory architecture, interrupt and exception handling, and external I/O.
JIT (Just-In-Time) Compilation
Compilation done during execution of a program – at run time – rather than prior to execution.
Kernel Bypass
Techniques that allow user-space applications to access network hardware directly, avoiding the overhead of the kernel's network stack for higher performance.
LKM (Loadable Kernel Module)
An object file that contains code to extend the running kernel (e.g., device drivers).
LTO (Link-Time Optimization)
A program optimization performed by a compiler at link time, allowing optimizations across different source files.
MMU (Memory Management Unit)
Hardware unit responsible for handling virtual memory access and address translation.
NASM (Netwide Assembler)
An assembler and disassembler for the Intel x86 architecture.
NUMA (Non-Uniform Memory Access)
A memory design used in multiprocessing, where the memory access time depends on the memory location relative to a processor.
PGO (Profile-Guided Optimization)
Compiler optimization technique that uses profiling data from test runs to inform optimization decisions.
Portage
The package management system used by Gentoo Linux.
POSIX Threads (Pthreads)
A POSIX standard for threads, defining a standard API for creating and manipulating threads.
RAII (Resource Acquisition Is Initialization)
A programming idiom (especially in C++ and Rust) where resource management (allocation, deallocation) is tied to object lifetime through constructors and destructors.
RPN (Reverse Polish Notation)
A mathematical notation in which operators follow their operands.
SIMD (Single Instruction, Multiple Data)
A class of parallel computers in Flynn's taxonomy. It describes computers with multiple processing elements that perform the same operation on multiple data points simultaneously.
Syscall (System Call)
The programmatic way in which a computer program requests a service from the kernel of the operating system it is executed on.
TUI (Text-based User Interface)
A user interface that works with text, as opposed to GUIs (Graphical User Interfaces).
USE Flags
Keywords in Gentoo's Portage system that define support for optional features when building packages.
Valgrind
An instrumentation framework for building dynamic analysis tools; Memcheck is its most famous tool for detecting memory errors.
VFS (Virtual File System)
An abstraction layer on top of a more concrete file system, allowing client applications to access different types of concrete file systems uniformly.
VTune
Intel's performance analysis tool for CPU and system profiling.
XDP (eXpress Data Path)
A high-performance data path within the Linux kernel for processing network packets.
